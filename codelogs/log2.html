Code Log 2: CSS Measuring Units

When working with CSS, we use units to measure how big we want certain elements to be. Without these, CSS would not know what it's supposed to mean, so they're pretty important. Theres a lot of units we can use, divded into two catagories; absolute, and relative.

But what is the difference between these catagories??

Good question, conveinent voice of the audience; that's the { pt } of this code log.

Absolute Units

An absolute unit represents a unit that will remain consistent no matter the size of your screen or any other factors. It's a conventional unit of measurement.

Examples

  • Pixels { px }
    • Pixels are one of the most common units developers use, and one of the smallest. A single pixel consists of only 1/96th of an inch.
  • Points { pt }
    • Points are slightly larger than pixels, consisting of 1/72nd of an inch.
  • Inch { in }
    • An inch is the unit of measurement that the common man is most familar with. Its length consists of 10/10ths of an inch.
  • Picas { pc }
    • Picas are a unit of measurement that I had never heard of before now, but they are 1/6th of an ich long. The more you know!

Relative Units

Relative units are used to scale an element's size relative to the page it is on. It scales to the width of the screen you are using, so it's very useful for mobile-destop compatability.

Examples

  • Viewport Width { vw }
    • Viewport width scales the width of the element to a given percent of the size of the viewport. { 1vw } would mean the element scales to 1% of the total length of the page.
  • Viewport Height { vh }
    • Viewport height is exactly like viewport width, but scaling for height instead. { 1vh } would scale an element to 1% of the height of the page.
